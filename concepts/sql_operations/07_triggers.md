# 트리거 (Database Triggers)

## 1. 트리거의 개념
- 데이터베이스의 이벤트에 반응하여 자동으로 실행되는 프로시저
- 데이터 무결성과 일관성을 유지하기 위한 메커니즘
- 업무 규칙을 데이터베이스 수준에서 강제

## 2. 트리거의 유형

### 2.1 실행 시점에 따른 분류
- BEFORE 트리거
- AFTER 트리거
- INSTEAD OF 트리거

### 2.2 이벤트에 따른 분류
- INSERT 트리거
- UPDATE 트리거
- DELETE 트리거

## 3. 트리거의 구성요소

### 3.1 기본 구성
- 트리거 이름
- 트리거 이벤트
- 트리거 시점
- 트리거 본문

### 3.2 특수 테이블
- NEW 테이블
- OLD 테이블
- 임시 테이블

## 4. 트리거 작성

### 4.1 기본 문법
- CREATE TRIGGER 구문
- 조건문 사용
- 예외 처리

### 4.2 고려사항
- 성능 영향
- 순환 참조
- 디버깅 방법

## 5. 트리거 활용

### 5.1 일반적인 용도
- 감사 로그 기록
- 파생 데이터 계산
- 참조 무결성 유지

### 5.2 업무 규칙 구현
- 데이터 검증
- 자동 값 설정
- 연계 처리

## 6. 트리거 관리

### 6.1 트리거 관리 작업
- 트리거 활성화/비활성화
- 트리거 수정
- 트리거 삭제

### 6.2 성능 최적화
- 실행 계획 분석
- 리소스 사용 모니터링
- 코드 최적화
